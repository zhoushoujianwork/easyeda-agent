## 三条战略结论

1. **架构方向已被官方「撞型」验证。** 官方 `eext-run-api-gateway` 的桥接模型
   （port-scan 49620-49629 / `/health` / WS 握手 / 自动重连）和我们几乎一模一样——
   自建 connector 而非用官方 gateway 的决策正确，基础设施层无需再纠结。

2. **PCB 智能布线有平台级天花板，往这砸是撞墙。** 单路避障、推挤、等长蛇形、扇出
   **无 API，agent 永远做不了**（survey §7.4 硬边界）。我们能给的上限：铺铜、按坐标
   布线/过孔、rip-up、约束定义、长度/skew 报告、DRC 门禁。

3. **真正的高价值前沿在两端，不在 PCB 布线中段：**
   - **上游 = agent 决策层**：器件选型(A8 LLM 两段式)、在线搜索缓存(A1 写回链路)、
     换封装/换符号(A6)。这是「agent 画板」区别于「人点按钮」的核心壁垒。
   - **下游 = 可制造性闭环**：BOM C 号补全已做但**还没接进 daemon 默认导出**
     （FEATURES 明确列为 follow-up）；网表 JSON 交叉校验(A7)能降 DRC 误报漏报。
